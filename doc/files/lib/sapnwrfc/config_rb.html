Configuration for an RFC connection can be passed in two distinct ways, but those two methods can be combined together.

(1) Config is loaded via a YAML based config file (2) Config is passed in the code directly into the connection (3) a combination of (1) and (2), where (2) overides (1) at run time.

YAML config file format: 

  ashost: ubuntu.local.net
  sysnr: "01"
  client: "001"
  user: developer
  passwd: developer
  lang: EN
  trace: 2

At connection time, any valid parameters can be added or over ridden: 

  conn = SAPNW::Base.rfc_connect(:user => 'developer', :passwd => 'developer')

Valid connection parameters are: 

  ashost
  dest - used in conjunction with sapnwrfc.ini file
  mshost
  group
  sysid
  msserv
  sysnr
  lang
  client
  user
  passwd
  trace
  tpname
  gwhost
  gwserv
  codepage
  x509cert
  extiddata
  extidtype
  mysapsso2
  mysapsso
  getsso2
  snc_mode
  snc_qop
  snc_myname
  snc_partnername
  snc_lib
